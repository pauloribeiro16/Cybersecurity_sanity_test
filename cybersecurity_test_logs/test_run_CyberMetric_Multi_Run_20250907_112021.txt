Buffer overflow attacks occur when a program writes more data to a buffer than it can hold, potentially overwriting adjacent memory and leading to arbitrary code execution. To mitigate such vulnerabilities, various security features have been developed.

- **Data Execution Prevention (DEP)** is designed specifically to mark certain regions of memory as non-executable. This prevents any code from being executed in those regions, thus thwarting attempts at executing malicious payloads injected via buffer overflow exploits.
  
- **Buffer Overflow Prevention (BOP)** is not a standard term used for specific security mechanisms. It generally refers to practices and technologies aimed at preventing buffer overflows but doesn't specify the method of marking memory as non-executable.

- **Intrusion Detection System (IDS)** monitors network or system activities for malicious actions or policy violations, alerting administrators to potential threats. It does not directly prevent code execution by controlling memory permissions.

- **Address Space Layout Randomization (ASLR)** randomizes the location of key data areas in a process's address space. While this makes it harder for attackers to predict where their injected code will execute, it does not mark any region as non-executable.
