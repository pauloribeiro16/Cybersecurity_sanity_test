To compare Web Application Firewalls (WAFs) and Runtime Application Self-Protection (RASP), we need to examine several key aspects: deployment location, detection methods, and their effectiveness in stopping zero-day attacks.

### Deployment Location

1. **Web Application Firewall (WAF):**
   - A WAF is typically deployed as a separate component at the network boundary, either in front of or behind the application server.
   - It acts as a proxy that filters HTTP/HTTPS traffic between users and web applications based on predefined security rules.

2. **Runtime Application Self-Protection (RASP):**
   - RASP is embedded within the application itself, often at runtime inside the application code or its runtime environment.
   - This allows it to monitor application behavior from the inside as it executes.

### Detection Method

1. **Web Application Firewall (WAF):**
   - WAFs use signature-based detection methods to identify known attack patterns and anomalies in HTTP requests.
   - They rely on a database of predefined rules or signatures that match known vulnerabilities or attack vectors.

2. **Runtime Application Self-Protection (RASP):**
   - RASP uses behavior-based detection, monitoring the application's actions at runtime.
   - It can detect suspicious activities by analyzing code execution and data flow within the application context.
   - RASP can potentially identify unusual patterns that do not match known signatures but indicate malicious behavior.

### Effectiveness Against Zero-Day Attacks

1. **Web Application Firewall (WAF):**
   - WAFs are less effective against zero-day attacks because they primarily rely on known signatures and rules. 
   - Since zero-day vulnerabilities exploit previously unknown security flaws, the WAF might not have the necessary signatures to detect such threats.

2. **Runtime Application Self-Protection (RASP):**
   - RASP is more capable of stopping zero-day attacks due to its behavior-based approach.
   - By analyzing application behavior and context in real-time, RASP can potentially identify anomalies that deviate from normal operations, even if they are not previously known vulnerabilities.

### Conclusion

**Which is Better at Stopping Zero-Day Attacks?**

RASP is generally better equipped for stopping zero-day attacks compared to WAFs. This advantage stems from its ability to analyze the application’s behavior in real-time and detect anomalies that could indicate a novel attack, even if they do not match any known signature or pattern.

In summary:

- **WAF** is deployed at the network boundary with signature-based detection.
- **RASP** is embedded within the application using behavior-based detection.
- For zero-day attacks, **RASP** offers better protection due to its runtime analysis capabilities.
